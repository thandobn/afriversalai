Onboard a corporate & create their cohort

Bringing a corporate client onto AfriversalAI follows one clear path: you register them here, an AfriversalAI admin approves the cohort, and their staff enrol with a unique access code that stamps every learner with a cohort-coded Member ID. Here's exactly how it works.

The onboarding journey — six steps
1
Register the corporate
Capture the company, intake, seat count and branding below. We generate a unique, provisional access code for the cohort.
You
2
Admin review & approval
The cohort lands in the AfriversalAI admin console as Pending. An admin verifies the deal and approves it — only then does the access code go live for enrolment.
AfriversalAI admin
3
Distribute the access code
Once approved, share the cohort's access code with the corporate's staff (e.g. via their L&D lead).
You
4
Staff enrol & get a Member ID
Each employee registers with the code. They're grouped into this cohort and automatically issued a cohort-stamped Member ID (e.g. ACME-2026-014).
Automatic
5
Track the cohort dashboard
Enrolment and progress for this cohort appear here in your portal and in the admin console — grouped under the cohort's access-code block.
Automatic
6
Certify
When a learner is assessed as demonstrated, their certificate carries their cohort Member ID — tying every credential back to this corporate intake.
Automatic
